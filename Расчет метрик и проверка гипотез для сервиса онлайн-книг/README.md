# Проект: расчет метрик и проверка гипотез для сервиса онлайн-книг
________________________________________________________
## Описание проекта
________________________________________________________
Работа с базой данных сервиса Яндекс.Книги. Состоит из нескольких этапов: с помощью SQL расчитываются метрики по запросу заказчика, с помощью Python - проверяются гипотезы и анализируются результаты A/B тестирования.

## Цели исследования 
_______________________________________________________
1. Расчет 5 метрик на основе данных в массиве с помощью SQL: MAU авторов в сервисе, MAU произведений в сервисе, Retention Rate, LTV, Средняя выручка прослушанного часа как аналог среднего чека (код и результаты представлены в файле "**Расчет метрик в SQL**")
2. Подготовка данных с помощью SQL для дальнейшего анализа: выделение в массиве данных информации, касающейся только Москвы и Санкт-Петербурга (код представлен в файле "**Подготовка данных в SQL**")
3. Проверка гипотезы о том, что пользователи из Санкт-Петербурга проводят в среднем больше времени за чтением и прослушиванием книг в приложении Yandex книги, чем пользователи из Москвы (код представлен в файле "**Гипотезы в Python**")
4. Проверка гипотезы о том, что новый интерфейс сайта приводит к статистически значимому величению доли пользователей, совершающих покупки (код представлен в файле "**Гипотезы в Python**")

## Инструменты
_______________________________________________________
SQL, Python (Pandas, Matplotlib, Seaborn, Numpy, Scipy)

## Результаты
_______________________________________________________
1. Рассчитаны 5 метрик по запросу заказчика
2. Массив данных ограничен информацией по 2 городам - Москве и Санкт-Петербургу
3. Для проверки результата теста использовались t-test Уэлча при уровне статистической значимости 5% (alpha=0.05). В результате анализа не удалось найти статистическое подтверждения альтернативной гипотезе о том, что петербургские пользователи проводят больше времени за прочитыванием и прослушиванием контента в приложении. Показатель p-value значительно превышает alpha, и нулевая гипотеза об отсутствии различий между двумя выборками не выявлена. Подобный результат может быть связан с особенностями выборок (московская выборка численно в 2.7 раза превышает петербургскую, и жители двух крупных городов на деле демонстрируют очень похожее поведение в контексте привычек к чтению и прослушиванию книг).
4. По результатам А/В-тестирования удалось установить, что конверсия в покупку среди тестовой группы, пользовавшейся версией сайта с обновленным интерфейсом, оказалась выше, чем в контрольной группе, пользовавшейся старой версией сайта. Однако положительная динамика составила всего 2,2% вместо ожидавшихся как минимум 3%. Статистический тест подтверждает значимость полученного результата. Таким образом, разработка нового интерфейса позволила добиться эффекта в изменении конверсии, однако он оказался недостаточным. В связи с этим рекомендуется рассмотреть возможность и других нововведений, которые были бы способны улучшить конверсию до желаемого уровня.
